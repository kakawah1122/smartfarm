## 使用建议
1. **优先阅读根目录现行文档**（如 `README.md`、`项目开发规范.md`）  
2. 如需回溯历史迭代、专项修复细节，再到归档目录检索  
3. 新增历史资料时，请保持命名清晰并更新本说明

如未来再次大量归档，请新建以日期命名的子目录（例如 `docs/archive/2026-03/`），并在本说明中追加批次记录。
