# 🔍 检查云函数超时配置

## 当前问题
AI识别一直在 **3秒** 超时，说明某个云函数的超时配置是3秒。

## 需要检查的云函数

### 1. recognizeGoosePrice （新建的，可能是问题所在）
**这个云函数可能还是默认的3秒超时！**

#### 检查步骤：
1. 打开微信开发者工具
2. 点击：云开发 → 云控制台
3. 左侧菜单：云函数
4. 找到 `recognizeGoosePrice` 云函数
5. 查看"超时时间"配置
6. **如果是3秒 → 改为 60秒 → 保存**

### 2. ai-multi-model （应该已经是60秒）
按同样步骤检查，确保也是60秒。

## ⚠️ 重要
- config.json 中的 timeout 只在**首次部署**时生效
- 后续修改必须在**云控制台**手动更改
- 新建的云函数默认是 **3秒** 超时

## 验证方法
配置完成后：
1. 不需要重新部署
2. 直接重新测试 AI 识别
3. 查看云函数日志，看是否执行到了 AI 调用部分

## 预期日志
正常情况下应该看到：
```
====== 开始AI识别鹅价 ======
图片fileID: cloud://...
🤖 调用AI模型识别...
⏱️  图片fileID: cloud://...
AI调用结果: ...
```

如果在"调用AI模型识别"后立即超时，说明是 recognizeGoosePrice 的配置问题。
