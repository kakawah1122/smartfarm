# 治疗记录表单字体大小统一优化

## 问题诊断

### 原始问题
药品卡片内的文字字体大小与表单其他区域不一致，主要表现为：
- **药品名称"鹅瘟清"** - 34rpx（过大）
- **标签"数量"、"用法"** - 32rpx（过大）
- **输入框内文字** - 32rpx（过大）
- **单位"件"** - 30rpx（过大）

### 标准字体大小
表单其他区域统一使用 **28rpx** 作为标准字体大小：
```scss
.form-label {
  font-size: 28rpx;  // ✅ 表单标签标准
}
```

## 字体大小规范总结

### 标准层级
本项目表单使用的字体大小标准：

| 元素类型 | 字体大小 | 用途 | 示例 |
|---------|---------|------|-----|
| **标题** | 28rpx | 区域标题（蓝色背景） | "用药管理"、"备注信息" |
| **标签** | 28rpx | 表单字段标签 | "数量"、"用法"、"诊断结果" |
| **输入框** | 28rpx | 用户输入文字 | 输入框内容、选择器显示 |
| **药品名** | 28rpx | 药品卡片标题 | "鹅瘟清"、"土霉素" |
| **单位** | 28rpx | 数量单位 | "件"、"克" |
| **次要信息** | 24-26rpx | 提示文字、辅助信息 | "库存: 89件"、"可选" |

### 设计原则
1. **一致性优先**：同类元素使用相同字体大小
2. **层级清晰**：通过字重、颜色而非字号区分重要性
3. **可读性**：28rpx 在小程序中是最佳可读字号
4. **减少变化**：避免使用过多字号造成视觉混乱
